## Responsive Behavior

### 80 Columns (Standard Terminal)
```
Full layout fits comfortably
Equipment grid: 3-column layout
Stats sidebar: Full display
All borders intact
```

### 70 Columns (Minimum Width)
```
Slightly compressed
Equipment grid: 2-column layout
Stats sidebar: Narrower but readable
Item names may truncate
```

### < 70 Columns (Fallback)
```
Single-column layout (not implemented in v1.0)
Equipment slots stack vertically
Stats move to bottom
Recommend horizontal scrolling
```

## Integration Examples

### Live Combat Display
```python
from rich.live import Live

with Live(render_paper_doll(char), refresh_per_second=4) as live:
    while combat_active:
        char.hp_current -= damage  # Update state
        live.update(render_paper_doll(char))  # Auto-refresh
        await asyncio.sleep(0.25)
```

### Side-by-Side Party View
```python
from rich.columns import Columns

party = [character1, character2, character3]
sheets = [render_paper_doll(c, console_width=60) for c in party]
console.print(Columns(sheets, equal=True))
```

## Performance Metrics

- Single render: ~5ms (Raspberry Pi 5)
- Live update (4 FPS): Minimal CPU (<5%)
- Discord embed JSON: <1ms
- Memory footprint: ~2KB per character
